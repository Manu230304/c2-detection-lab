# Wazuh Setup — Manager & Agent

## Wazuh Manager (Ubuntu Server)

### Installation (single-node, all-in-one)

```bash
curl -sO https://packages.wazuh.com/4.7/wazuh-install.sh
sudo bash wazuh-install.sh -a
```

This installs:
- Wazuh Manager
- Wazuh Indexer (OpenSearch)
- Wazuh Dashboard

Default dashboard credentials are printed at the end of the install. Access via `https://<server-ip>`.

### Verify services

```bash
sudo systemctl status wazuh-manager
sudo systemctl status wazuh-indexer
sudo systemctl status wazuh-dashboard
```

### Adding custom rules

Custom rules go in `/var/ossec/etc/rules/local_rules.xml`.  
After editing, restart the manager:

```bash
sudo systemctl restart wazuh-manager
```

## Wazuh Agent (Windows 11)

### Deployment

From the Wazuh dashboard:
1. Go to **Agents → Deploy New Agent**
2. Select Windows, enter the Manager IP
3. Copy and run the generated PowerShell command on the Windows VM as Administrator

```powershell
# Example (auto-generated by dashboard)
Invoke-WebRequest -Uri https://<manager-ip>/packages/wazuh-agent.msi -OutFile wazuh-agent.msi
Start-Process msiexec.exe -ArgumentList '/i wazuh-agent.msi /quiet WAZUH_MANAGER="<manager-ip>"' -Wait
NET START WazuhSvc
```

### Verify agent connection

```cmd
NET START | findstr Wazuh
```

On the Manager, the agent should appear as **Active** in the dashboard within 30 seconds.

## Sysmon Integration with Wazuh

Sysmon events are automatically collected by the Wazuh Agent from the Windows Event Log channel `Microsoft-Windows-Sysmon/Operational`.

Ensure `ossec.conf` on the agent includes:

```xml
<localfile>
  <location>Microsoft-Windows-Sysmon/Operational</location>
  <log_format>eventchannel</log_format>
</localfile>
```

This is typically included by default in modern Wazuh agent deployments.
